# Article Writing Standard

用于 `article-site` 的新闻与分析稿写作规范。

## 1. 目标

文章不是把信息堆在一起，而是把读者从「发生了什么」带到「为什么重要」。

每篇文章都应满足三件事：

1. **结构清楚**：读者能一眼看懂文章怎么展开。
2. **Markdown 正常**：标题、次级标题、列表、链接都符合 Markdown 规范。
3. **逻辑顺滑**：段落之间有推进关系，而不是条目堆砌。

---

## 2. 标准结构

每篇文章默认采用以下骨架：

```md
# 标题

导语：用 1~2 段交代这篇文章为什么值得读。

## 今日重点 / Key takeaways

- 重点 1
- 重点 2
- 重点 3

## 为什么这件事重要

用 1~2 段说明它和行业、政策、产品、资本、用户体验之间的关系。

## 1. 第一条核心信息

### 发生了什么

### 为什么值得关注

### 接下来观察什么

## 2. 第二条核心信息

### 发生了什么

### 为什么值得关注

### 接下来观察什么

## 结语 / Bottom line

用 1 段把全文收住，给读者一个明确判断。

## 来源 / Sources

- 来源 A
- 来源 B
```

---

## 3. Markdown 规则

### 必须做到

- 文章主标题只用一个 `#`
- 一级分节统一用 `##`
- 分节内的小节统一用 `###`
- 列表必须用 `- ` 或 `1.` 这种标准 Markdown 写法
- 链接直接写完整 URL 或 `[标题](链接)`
- 段落之间必须空一行

### 不要再出现

- 一整段把 `Source / Published / Summary / Why it matters` 生硬拼在一起
- 没有标题层级、全文像纯文本堆叠
- 同一篇里中英文格式风格完全不同
- 句子过长，读者一眼看不到重点

---

## 4. 表达规则

### 导语

导语不要重复标题，而要回答：

- 这篇文章到底在谈什么
- 为什么今天值得看
- 读完会得到什么判断

### 正文

每个部分都要有推进关系：

1. 先说事实
2. 再说意义
3. 最后说后续观察点

### 结尾

结尾不是重复摘要，而是要回答：

- 今天真正的信号是什么
- 这件事会把局面推向哪里

---

## 5. 风格要求

### 应该有的感觉

- 像编辑整理过，而不是机器把 RSS 拼起来
- 有判断，但不过度夸张
- 有节奏，段落短，读起来顺
- 重点靠结构和句子推进，不靠堆术语

### 避免的感觉

- 通稿腔
- 关键词堆砌
- 标题党
- 机械重复“这很重要，因为这很重要”

---

## 6. 双语要求

- `contentEn` 与 `contentZhHant` 都必须是完整文章，不是半翻译状态
- 两种语言应保持相同结构
- 可以自然表达，不要求逐句硬翻
- 中文标题层级、标点、段落节奏要符合中文阅读习惯

---

## 7. 快速检查清单

发布前至少过一遍：

- [ ] 有 `#` 主标题
- [ ] 有 `##` 分节标题
- [ ] 有 `###` 说明性小标题
- [ ] 导语不是废话
- [ ] 列表是标准 Markdown
- [ ] 每个核心条目都包含“发生了什么 / 为什么重要 / 接下来观察什么”
- [ ] 结尾有明确判断
- [ ] 来源区清楚可追溯

---

## 8. 一句话原则

**不要把新闻条目贴成文章，要把新闻整理成判断。**
